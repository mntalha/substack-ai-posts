3
+ Positional Encoding — How It’s Computed
(3, 4, 512) + (4, 512) → (3, 4, 512)

The PE matrix is computed once from the formulas below, not learned. The same PE matrix is added to all 3 sentences in the batch.

// For each position pos and dimension index i:
PE[pos, 2i]   = sin(pos / 100002i/512)
PE[pos, 2i+1] = cos(pos / 100002i/512)

// Example: pos=1, dim=0 (i=0):
PE[1, 0] = sin(1 / 100000/512) = sin(1 / 1) = sin(1.0) = 0.8415
PE[1, 1] = cos(1 / 100000/512) = cos(1.0) = 0.5403

// Example: pos=1, dim=2 (i=1):
PE[1, 2] = sin(1 / 100002/512) = sin(1 / 1.018) = sin(0.9823) = 0.8280
PE[1, 3] = cos(1 / 100002/512) = cos(0.9823) = 0.5607

// Example: pos=1, dim=510 (i=255, highest freq pair):
PE[1, 510] = sin(1 / 10000510/512) = sin(0.000102) ≈ 0.0001
PE[1, 511] = cos(1 / 10000510/512) = cos(0.000102) ≈ 1.0000

Key insight: frequency varies by dimension

Low dims (0, 1): frequency ≈ 1.0 → oscillates rapidly. Distinguishes adjacent tokens.
High dims (510, 511): frequency ≈ 0.0001 → barely changes. Encodes coarse position.
Together, every position gets a unique fingerprint. The same PE row is added across all 3 batches.

⤷ Input to Step 4: Position-aware embeddings X (3, 4, 512) — this X is also saved as Xinput for the residual connection in Step 9
4
Project Q, K, V — What Are the W Matrices?
(3, 4, 512) × W → 3 × (3, 4, 512)

Three learned weight matrices transform the input into three different “views.” Each W is 512×512 — all 8 heads packed into one matrix.

MatrixShapeWhat It DoesAnalogy
WQ (Query) (512, 512) Transforms each token into “what am I looking for?” “cat” asks “who is my verb?”
WK (Key) (512, 512) Transforms each token into “what do I advertise?” “sat” advertises “I’m a verb”
WV (Value) (512, 512) Transforms each token into “what info do I share?” “sat”’s meaning content
WO (Output) (512, 512) Mixes concatenated head outputs back together Merges 8 perspectives into one
Q = X @ WQ   // (3,4,512) × (512,512) → (3,4,512)
K = X @ WK   // (3,4,512) × (512,512) → (3,4,512)
V = X @ WV   // (3,4,512) × (512,512) → (3,4,512)

// WQ, WK, WV are SHARED across all positions and all batches.
// The same WQ transforms “The”, “cat”, “sat”, “down” — AND all 3 sentences.
// Params per attention sub-layer: 4 × 512 × 512 = 1,048,576
Why 512×512 and not 512×64?

The paper packs all 8 heads into one big matrix for efficiency. WQ ∈ ℝ512×512 is equivalent to 8 separate WiQ ∈ ℝ512×64 stacked side by side. One large matmul instead of 8 small ones — same math, much faster.

Why reshape into chunks?

1. Multiple perspectives: Each head runs attention over 64 dims independently. Head 1 learns “subject-verb.” Head 5 learns “adjective-noun.” One big head averages everything — 8 small heads specialize.
2. Same total cost: 8 × (4×64)×(64×4) = 1 × (4×512)×(512×4). Identical computation, just partitioned.
3. GPU efficiency: Reshape is free (no data copy, just a view change). The 8 heads become a batch-like dimension — one fused kernel.

Full shape: (3, 8, 4, 64) — batch=3, heads=8, tokens=4, dim=64. Same for Q, K, and V.

What is WO and why is it needed?

After concatenation, each token has 512 dims — but the information is still siloed by head. Dims 0–63 came from Head 1, dims 64–127 from Head 2, etc. They never interacted.

WO ∈ ℝ512×512 is the only matrix that allows cross-head communication. It learns to blend features discovered by different heads: Head 1 found “cat is the subject,” Head 3 found “sat is past tense” → WO combines them into one unified 512-d representation.

Without WO, multi-head attention would just be 8 independent single-head attentions glued together — no synergy. WO is what makes the “multi” in multi-head actually useful.

Output shape: (3, 4, 512) — same dimensionality as the input X from Step 3, which is essential for the residual connection next.

⤷ Input to Step 10: Normalized residual (3, 4, 512) — each token now has context from attention + its original embedding, normalized

Feed-Forward Network (per token)
(3, 4, 512) → (3, 4, 2048) → (3, 4, 512)
FFN(x) = max(0, x @ W₁ + b₁) @ W₂ + b₂
// W₁: (512, 2048) — expand 4×
// W₂: (2048, 512) — compress back
// Applied independently to each of 3×4 = 12 tokens

ReLU zeros ~half the 2048 neurons. Another residual + LayerNorm. ~2.1M params per FFN. Shape: (3, 4, 512).

What Gets Reused vs. Recreated Each Layer?
Critical detail

Each of the 6 encoder layers runs steps 4–10, but they do NOT share weights:

ComponentReused or New?Explanation
WQ, WK, WV, WO NEW per layer Layer 1 has WQ1, layer 2 has WQ2. 6 × 4 = 24 attention weight matrices total.
W₁, b₁, W₂, b₂ (FFN) NEW per layer Each layer’s FFN has its own weights. 6 × 2.1M = 12.6M FFN params.
LayerNorm γ, β NEW per layer 12 LayerNorms total (2 per layer × 6). Small: 1024 params each.
Q, K, V tensors RECOMPUTED Activations. Each layer takes the previous output as X, computes fresh Q, K, V.
Attention scores RECOMPUTED Each layer produces fresh (3, 8, 4, 4) maps. Layer 1: syntax. Layer 6: semantics.
Wemb USED ONCE (step 2) Only at the start. Not touched during the 6 layers.
Positional Encoding USED ONCE (step 3) Added once before layer 1. Not re-added later (unlike RoPE in modern models).
Residual stream shape SAME throughout Every layer: (3, 4, 512) in, (3, 4, 512) out. This makes stacking possible.
x = embed(tokens) + PE // (3, 4, 512) — done once

for layer in [1, 2, 3, 4, 5, 6]:
  Q = x @ WQlayer // this layer’s weights — NOT shared
  K = x @ WKlayer
  V = x @ WVlayer
  attn = softmax(QKT/√64) @ V
  attn_out = concat(attn) @ WOlayer
  x = LNlayer,1(x + attn_out) // residual
  x = LNlayer,2(x + FFNlayer(x)) // residual

// Encoder output: (3, 4, 512)
Why not share weights across layers?

Each layer sees a progressively more refined representation. Layer 1 learns basic patterns from raw embeddings. Layer 6 learns abstract relationships from 5 rounds of attention. Sharing would force identical transformations — the paper tested it and performance dropped.
