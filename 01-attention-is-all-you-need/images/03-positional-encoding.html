Positional Encoding

How the Transformer knows word order without recurrence

PE(pos, 2i) = sin(pos / 100002i/dmodel)
PE(pos, 2i+1) = cos(pos / 100002i/dmodel)

pos = position in sequence (0 to S-1) · i = dimension index (0 to d_model/2 - 1) · d_model = 512

← Dimension index (0 to 511) →

↑ Position in sequence (0 to 63) ↓

Why Not Learned Embeddings?

The paper tested both. Results were nearly identical (Table 3, row E). They chose sinusoidal because it can extrapolate to longer sequences than seen during training. Learned embeddings are fixed to the max training length.

Why Sinusoids Work

Each dimension is a sinusoid with a different frequency. Low dimensions oscillate fast (short wavelength), high dimensions oscillate slowly. Together they create a unique fingerprint for each position. For any fixed offset k, PEpos+k is a linear function of PEpos — so the model can learn relative position attention.

How It's Used

Positional encodings are added (not concatenated) to the token embeddings. Both have dimension d_model = 512. The sum goes through dropout before entering the first encoder/decoder layer. This is done once — not at every layer.

Interview Question

Q: Why add instead of concatenate positional encodings?
A: Concatenation would increase d_model (512 → 1024), doubling compute in every attention and FFN layer. Addition keeps d_model at 512. The model learns to use the combined signal — empirically it works just as well.

Modern Alternatives

RoPE (Rotary Position Embedding) — used in LLaMA, Mistral. Encodes position by rotating Q and K vectors.
ALiBi — adds a linear bias to attention scores based on distance. No extra parameters.
