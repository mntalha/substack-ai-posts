Encoder Input
▶ ① Source Token IDs (1, 10)

The raw input sentence is tokenized (BPE) into integer IDs.

PositionTokenID
0The1996
1cat2368
2sat3940
………
9.4

Output: (1, 10) — batch=1, seq_len=10 integer tensor.

↓
▶ ② Token Embedding (1, 10) → (1, 10, 512)

Each token ID indexes into a learned matrix E ∈ ℝ37000 × 512.

embed = E[token_ids] # (1, 10, 512)

Each of the 10 tokens is now a 512-dim vector. Tokens have no position info yet — "cat" at position 1 and "cat" at position 7 would be identical.

Parameters: 37,000 × 512 = 18.9M

Output: (1, 10, 512)

↓ element-wise +
▶ ③ + Positional Encoding (1, 10, 512) → (1, 10, 512)

Sinusoidal position vectors are added element-wise to the embeddings. Not concatenated, not a separate layer — a pure sum.

PE(pos, 2i) = sin(pos / 10000^(2i/512)) PE(pos, 2i+1) = cos(pos / 10000^(2i/512)) x = embed + PE # (1, 10, 512)

This happens once, before layer 1. It is NOT reapplied between layers.

Why add, not concat? Adding keeps d_model = 512 unchanged. The PE magnitude is small relative to the embedding — it nudges the vector to encode position without dominating the learned content.

Parameters: 0 (sinusoids are fixed, not learned)

Output: (1, 10, 512) — this is x, the input to encoder layer 1.

↓ dropout(0.1)
Encoder Layer (× 6)
▶ ④ Q, K, V Linear Projections (1, 10, 512) → 3 × (1, 10, 512)

Three independent learned weight matrices project the same input x into queries, keys, and values:

Q = x @ W_Q # (1, 10, 512) @ (512, 512) → (1, 10, 512) K = x @ W_K # (1, 10, 512) @ (512, 512) → (1, 10, 512) V = x @ W_V # (1, 10, 512) @ (512, 512) → (1, 10, 512)
MatrixShapePurpose
WQ512 × 512What each token looks for
WK512 × 512What each token advertises
WV512 × 512What each token shares when attended to
These are plain linear layers (no activation, no ReLU). They are not the positional encoding — PE is already baked into x. The projections learn to extract different "aspects" of each token's representation.

Parameters: 3 × 512 × 512 = 786K (+ biases in the original paper)

Output: Q, K, V each (1, 10, 512)

↓ reshape
▶ ⑤ Reshape into 8 Heads 3 × (1, 10, 512) → 3 × (1, 8, 10, 64)

Each 512-dim vector is split into 8 chunks of 64 dims, then transposed so the head dimension comes before the sequence dimension:

Q = Q.view(1, 10, 8, 64).transpose(1, 2) # → (1, 8, 10, 64) K = K.view(1, 10, 8, 64).transpose(1, 2) # → (1, 8, 10, 64) V = V.view(1, 10, 8, 64).transpose(1, 2) # → (1, 8, 10, 64)

This is a free operation — no parameters, just reshaping memory. Each head now operates on its own 64-dim slice independently.

Why reshape, not 8 separate W matrices? Mathematically identical, but one big (512×512) matmul followed by a reshape is much faster on GPUs than 8 separate (512×64) matmuls. Same result, better hardware utilization.

Parameters: 0 (reshape only)

Output: Q, K, V each (1, 8, 10, 64)

↓
▶ ⑥ Scaled Dot-Product: QKT / √64 (1, 8, 10, 64) × (1, 8, 64, 10) → (1, 8, 10, 10)

Every token's query is compared against every token's key via dot product, then scaled:

scores = (Q @ K.transpose(-2, -1)) / √64 # (1, 8, 10, 10)

The shape (1, 8, 10, 10) means: for each of 8 heads, a 10×10 matrix where entry (i, j) = "how much should token i attend to token j."

DimensionSizeMeaning
Batch1One sentence
Heads88 independent perspectives
Rows (queries)10Each source token asking
Cols (keys)10Each source token answering
Why ÷ √64? Without scaling, dot products of 64-dim vectors have variance ≈ 64. Large values push softmax into saturation (near 0 or 1 outputs) where gradients vanish. Dividing by √64 = 8 normalizes variance back to ~1.

Output: (1, 8, 10, 10) — raw attention scores

↓ softmax(dim=-1)
▶ ⑦ Softmax → Attention Weights (1, 8, 10, 10) → (1, 8, 10, 10)

Each row of the 10×10 score matrix is turned into a probability distribution:

attn_weights = softmax(scores, dim=-1) # (1, 8, 10, 10) # each row sums to 1.0

Row i is the attention distribution for token i — how much weight it places on each other token.

Example for head 3, token "cat" (row 1):

Thecatsat….
cat →0.050.080.62…0.01

"cat" puts 62% of its attention on "sat" — it found its verb.

Output: (1, 8, 10, 10) — each row sums to 1

↓ × V
▶ ⑧ Attention × Values → Per-Head Output (1, 8, 10, 10) × (1, 8, 10, 64) → (1, 8, 10, 64)

The attention weights select and blend value vectors:

head_out = attn_weights @ V # (1, 8, 10, 10) @ (1, 8, 10, 64) → (1, 8, 10, 64)

Each token's output in a head is a weighted sum of all value vectors, using the attention weights from the previous step.

Token "cat" in head 3 gets: 0.62 × V("sat") + 0.08 × V("cat") + 0.05 × V("The") + …

Output: 8 heads × (1, 10, 64) = (1, 8, 10, 64)

↓ concat heads
▶ ⑨ Concat Heads + WO Projection (1, 8, 10, 64) → (1, 10, 512) → (1, 10, 512)

Step 1: Concat. The 8 head outputs are concatenated along the last dimension:

concat = head_out.transpose(1,2).reshape(1, 10, 512) # (1, 10, 512) # dims 0-63 = head 0, dims 64-127 = head 1, …, dims 448-511 = head 7

Problem: The 512 dimensions are siloed by head. Head 0's features occupy dims 0–63, head 1 has 64–127, etc. No cross-head information flow yet.

Step 2: WO projection. The output matrix blends across heads:

attn_output = concat @ W_O # (1, 10, 512) @ (512, 512) → (1, 10, 512)
WO is the only cross-head communication matrix. Without it, multi-head attention would be 8 independent attentions that never share information. WO lets the model combine "head 3 found the subject" with "head 5 found tense" into one coherent representation.

Parameters: WO = 512 × 512 = 262K

Total attention params per layer: WQ + WK + WV + WO = 4 × 262K = ~1.05M

Output: (1, 10, 512)

↓ + x (residual)
▶ ⑩ Residual + LayerNorm (1, 10, 512) → (1, 10, 512)

The attention output is added back to the original input (residual connection), then normalized:

h = LayerNorm(x + attn_output) # (1, 10, 512)

Residual: The sub-layer only needs to learn the delta — what to add. The original signal x is preserved.

LayerNorm: Normalizes across the 512-dim feature axis for each token independently. Mean → 0, variance → 1, then scale/shift with learned γ, β.

Why LayerNorm, not BatchNorm? Sequences have different lengths within a batch. BatchNorm normalizes across the batch — problematic with variable padding. LayerNorm normalizes each token's features independently.

Parameters: γ, β each 512 = 1K

Output: (1, 10, 512) — this is h

↓
▶ ⑪ Feed-Forward Network (position-wise) (1, 10, 512) → (1, 10, 2048) → (1, 10, 512)

A two-layer MLP applied independently to each token (same weights, per position):

FFN(h) = max(0, h @ W₁ + b₁) @ W₂ + b₂ Step 1: h @ W₁ + b₁ # (1, 10, 512) @ (512, 2048) → (1, 10, 2048) Step 2: ReLU # (1, 10, 2048) — kills negative values Step 3: × W₂ + b₂ # (1, 10, 2048) @ (2048, 512) → (1, 10, 512)
WeightShapeParameters
W₁512 × 20481,048,576
b₁20482,048
W₂2048 × 5121,048,576
b₂512512
Why expand 4×? The 2048-dim inner layer is a "memory bank." The expansion gives more capacity to process patterns; ReLU selectively activates ~half the neurons per input. The compression back to 512 forces the network to distill what's useful.

Parameters: ~2.1M per FFN

Output: (1, 10, 512)

↓ + h (residual)
▶ ⑫ Residual + LayerNorm (1, 10, 512) → (1, 10, 512)
output = LayerNorm(h + FFN(h)) # (1, 10, 512)

Same pattern as step ⑩. This output becomes the input to the next encoder layer.

One encoder layer total: ~3.15M params (1.05M attention + 2.1M FFN + ~1K norms)

Output: (1, 10, 512) → feeds into layer 2, 3, …, 6

↻ Repeat steps ④–⑫ for layers 2, 3, 4, 5, 6 — each reads (1, 10, 512), outputs (1, 10, 512)
↓ encoder output
▶ ⑬ Encoder Final Output (1, 10, 512)

After 6 layers, the encoder produces a rich, context-aware representation of every input token.

Token "cat" is no longer just "cat" — it encodes "cat as the subject of sat, which is modified by down, preceded by The, in a sentence that ends with a period."

This tensor is sent to every decoder layer as K and V for cross-attention. It is computed once and reused.

Total encoder params: 6 × 3.15M + 18.9M (embedding) ≈ 37.8M

Output: (1, 10, 512)

Decoder Input
▶ ⑭ Target Token IDs (shifted right) (1, 8)

During training: the correct target sequence, shifted right by 1 (so position 0 = <BOS>).

During inference: generated tokens so far + new token being predicted.

Output: (1, 8)

↓
▶ ⑮ Target Embedding + Positional Encoding (1, 8) → (1, 8, 512)

Same process as encoder: embedding lookup + sinusoidal PE addition.

y = Embedding(target_ids) + PE[:8] # (1, 8, 512)

The embedding matrix is shared with the encoder (weight tying).

Output: (1, 8, 512) — this is y

↓ dropout(0.1)
Decoder Layer (× 6)
▶ ⑯ Masked Multi-Head Self-Attention (1, 8, 512) → (1, 8, 512)

Identical to encoder self-attention (steps ④–⑨), but with a causal mask.

Q = y @ W_Q K = y @ W_K V = y @ W_V # all from y scores = Q @ K.T / √64 # (1, 8, 8, 8) scores = scores.masked_fill(mask, -∞) # upper triangle → -∞ attn = softmax(scores) @ V # (1, 8, 8, 64) → concat + W_O → (1, 8, 512)

The mask is a lower-triangular matrix: token i can only see tokens 0…i. After softmax, e-∞ = 0.

pos 0✓✗✗✗✗
pos 1✓✓✗✗✗
pos 2✓✓✓✗✗
…✓✓✓…✗
pos 7✓✓✓✓✓

Attention score shape: (1, 8, 8, 8) — 8 heads, each 8×8 (Ttgt × Ttgt)

Parameters: ~1.05M (same as encoder attention)

Output: (1, 8, 512)

↓ + y (residual) → LayerNorm
▶ ⑰ Cross-Attention (Decoder → Encoder) Q: (1, 8, 512) | K,V: (1, 10, 512) → (1, 8, 512)

This is where the encoder and decoder meet. The decoder asks: "given what I've generated so far, what parts of the input should I attend to?"

Q = decoder_h @ W_Q # (1, 8, 512) — from decoder's current state K = encoder_out @ W_K # (1, 10, 512) — from encoder final output V = encoder_out @ W_V # (1, 10, 512) — from encoder final output # Reshape to heads: Q → (1, 8, 8, 64) # 8 heads, 8 target positions, 64 dims K → (1, 8, 10, 64) # 8 heads, 10 source positions, 64 dims V → (1, 8, 10, 64) scores = Q @ K.T / √64 # (1, 8, 8, 64) @ (1, 8, 64, 10) → (1, 8, 8, 10) attn = softmax(scores) @ V # (1, 8, 8, 10) @ (1, 8, 10, 64) → (1, 8, 8, 64) → concat + W_O → (1, 8, 512)
Key shape: (1, 8, 8, 10) — for each of 8 heads, every decoder position (8) has a probability distribution over every encoder position (10). This is the "bridge" between input and output.

No causal mask here — each decoder token can see all encoder positions.

Parameters: ~1.05M (separate WQ, WK, WV, WO — not shared with self-attention)

Output: (1, 8, 512)

↓ + residual → LayerNorm
▶ ⑱ Feed-Forward Network (1, 8, 512) → (1, 8, 2048) → (1, 8, 512)

Same architecture as encoder FFN (step ⑪): 512 → 2048 → ReLU → 512.

Applied independently to each of the 8 target positions.

Parameters: ~2.1M

Output: (1, 8, 512)

↓ + residual → LayerNorm
▶ ⑲ Decoder Layer Output (1, 8, 512)

One decoder layer has 3 sub-layers:

Sub-layerParams
Masked Self-Attention (WQ, WK, WV, WO)~1.05M
Cross-Attention (WQ, WK, WV, WO)~1.05M
FFN (W₁, W₂)~2.1M

Total per decoder layer: ~4.2M — 50% more than an encoder layer because of the extra cross-attention.

Output: (1, 8, 512) → feeds into next decoder layer

↻ Repeat steps ⑯–⑲ for decoder layers 2, 3, 4, 5, 6
Output
↓ final decoder output
▶ ⑳ Linear Projection to Vocabulary (1, 8, 512) → (1, 8, 37000)

The final decoder output is projected to vocabulary size:

logits = decoder_out @ W_vocab # (1, 8, 512) @ (512, 37000) → (1, 8, 37000)

Each of the 8 target positions gets a score for every word in the vocabulary.

Weight tying: Wvocab is the transpose of the embedding matrix ET. The same 512-dim vectors that encode tokens as inputs also decode them as outputs. This saves 18.9M parameters and provides a useful inductive bias.

Parameters: 0 (shared with embedding)

Output: (1, 8, 37000) — raw logits (unnormalized scores)

↓ softmax
▶ ㉑ Softmax → Token Probabilities (1, 8, 37000) → (1, 8, 37000)

Each position's 37,000 scores become a probability distribution:

probs = softmax(logits, dim=-1) # (1, 8, 37000) # each of the 8 positions sums to 1.0

During training: compare against true target via cross-entropy loss (all 8 positions in parallel).

During inference: use decoding strategy (greedy / beam search / top-p sampling) on position T to pick the next token.

Output: (1, 8, 37000) — probability over vocabulary per position
