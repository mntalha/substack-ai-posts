Attention Mechanism — Step by Step

How information flows through Scaled Dot-Product and Multi-Head Attention

Scaled Dot-Product Attention

Attention(Q, K, V) = softmax(QKT / √dk) V
  1. Input Q, K, V matrices
    Q: (B, 8, S, 64) — what each token is looking for
    K: (B, 8, S, 64) — what each token advertises
    V: (B, 8, S, 64) — what each token provides
  2. Compute QKT
    Matrix multiply Q × KT
    (B, 8, S, 64) × (B, 8, 64, S) → (B, 8, S, S)
    Result: pairwise similarity scores between all tokens
  3. Scale by √dk = √64 = 8
    Divide every score by 8
    (B, 8, S, S) — prevents softmax saturation
  4. Apply mask (optional)
    For decoder self-attention: set future positions to -∞
    For encoder self-attention: no mask needed
  5. Softmax along last dimension
    Each row becomes a probability distribution
    (B, 8, S, S) — rows sum to 1.0
  6. Multiply by V
    (B, 8, S, S) × (B, 8, S, 64) → (B, 8, S, 64)
    Result: weighted sum of values for each token

Why scale by √dk? When dk = 64, dot products have variance ≈ 64. Values like 50 or -50 push softmax into regions where gradients ≈ 0. Dividing by 8 brings variance to ≈ 1, keeping gradients healthy.

Multi-Head Attention

MultiHead(Q, K, V) = Concat(head1, …, headh) WO
headi = Attention(QWiQ, KWiK, VWiV)
  1. Start with input X
    (B, S, 512) — one vector per token
  2. Project Q, K, V with separate learned weights
    WiQ, WiK, WiV ∈ ℝ512×64 per head
    Total: (B, S, 512) → 3 × (B, 8, S, 64)
  3. Run 8 attention heads in parallel
    h₁
    64d
    h₂
    64d
    h₃
    64d
    h₄
    64d
    h₅
    64d
    h₆
    64d
    h₇
    64d
    h₈
    64d
    Each head: (B, S, 64) output
  4. Concatenate all heads
    ⊕
    8 × (B, S, 64) → (B, S, 512)
  5. Final projection WO
    WO ∈ ℝ512×512
    (B, S, 512) → (B, S, 512)

Same cost as single-head. 8 heads × 64 dims = 512. One big head at 512d would have the same FLOPs. But 8 heads learn 8 different relationship patterns — syntax, semantics, position, etc.

Three Uses of Attention in the Transformer

Type Q comes from K, V come from Score shape Mask? Purpose
Encoder Self-Attention Encoder layer Same encoder layer (B, 8, S, S) No Each input token attends to all input tokens
Masked Decoder Self-Attention Decoder layer Same decoder layer (B, 8, T, T) Yes — causal Each output token attends to previous outputs only
Cross-Attention Decoder layer Encoder output (B, 8, T, S) No Decoder looks at the entire encoded input

Complexity Comparison

Layer Type Complexity per Layer Sequential Operations Max Path Length
Self-Attention O(S² · d) O(1) O(1)
Recurrent (RNN) O(S · d²) O(S) O(S)
Convolutional O(k · S · d²) O(1) O(logk(S))

S = sequence length · d = model dimension · k = kernel size. Self-attention wins on sequential ops and path length. The S² cost is why long sequences need FlashAttention.
